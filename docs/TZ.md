# Техническое задание (ТЗ)

## Проект: Симулятор STM32F103C8T6 (Blue Pill) + микросервис управления

Версия документа: 1.0  
Статус: Draft (MVP scope defined)

---

# 0. Термины и определения

- **SoC / MCU** — микроконтроллер STM32F103C8T6.
- **Симулятор** — программа, исполняющая бинарный код STM32 в виртуальной модели памяти и периферии.
- **CPU Core** — модель ARM Cortex-M3 (архитектура ARMv7-M, Thumb/Thumb-2).
- **MMIO** — memory-mapped I/O.
- **NVIC** — Nested Vectored Interrupt Controller.
- **KeyDB** — брокер задач (совместим с Redis).
- **OTel** — OpenTelemetry.
- **MVP** — минимально жизнеспособная версия.
- **IRQ** — аппаратное прерывание.
- **Exception** — архитектурное исключение ARMv7-M.

---

# 1. Общие положения

## 1.1 Назначение проекта

Разработать программную систему, состоящую из:

1. Симулятора STM32F103C8T6 (C)
2. Микросервиса управления (Go)

Система должна исполнять прошивки, моделировать периферию и обеспечивать удалённый запуск задач.

---

## 1.2 Цель

Создать воспроизводимую, тестируемую и расширяемую платформу для симуляции STM32F103C8T6 с возможностью автоматизированного запуска прошивок.

---

## 1.3 Основные задачи

1. Реализовать CPU core ARMv7-M.
2. Реализовать NVIC.
3. Реализовать TIM2.
4. Реализовать USART1.
5. Организовать модель памяти.
6. Реализовать микросервис обработки задач.
7. Обеспечить трассировку через OpenTelemetry.
8. Обеспечить unit и integration тестирование.
9. Подготовить документацию.

---

# 2. Архитектура системы

Система состоит из:

- C-симулятора
- Go-микросервиса
- KeyDB
- OpenTelemetry backend

Общий поток:

1. Задача помещается в KeyDB.
2. Микросервис извлекает задачу.
3. Микросервис запускает симулятор.
4. Симулятор выполняет прошивку.
5. Результат возвращается в KeyDB.
6. Логирование фиксируется через OTel.

---

# 3. Требования к симулятору (C)

## 3.1 Модель памяти

Реализовать области:

- Flash: 0x08000000
- SRAM:  0x20000000
- MMIO:  0x40000000
- SCS:   0xE000E000

### Reset sequence

```
MSP = *(uint32_t*)FLASH_BASE
PC  = *(uint32_t*)(FLASH_BASE + 4)
```

Поддержка VTOR — желательна.

---

## 3.2 CPU Core

### Архитектура

- ARMv7-M
- Thumb / Thumb-2
- 16 и 32-бит инструкции

### Обязательный набор инструкций (MVP)

- ADD, SUB, MOV, CMP
- AND, ORR, EOR
- LDR, STR
- PUSH, POP
- B, BL, BX
- SVC
- NOP

### Обязательные элементы

- APSR (N, Z, C, V)
- Fetch → Decode → Execute
- Детекция неподдержанной инструкции
- Контролируемое завершение

### Требования к состоянию CPU

Структура состояния должна содержать:

- R0–R12
- MSP
- PSP (опционально)
- LR
- PC
- xPSR
- PRIMASK
- BASEPRI (опционально)

---

## 3.3 Исключения и NVIC

Реализовать:

- Exception entry/return
- Стековый фрейм Cortex-M
- ISER / ICER
- ISPR / ICPR
- IPR (минимум 4 старших бита)
- Выбор IRQ по приоритету

Обязательно поддержать:

- Reset
- HardFault
- SVC
- TIM2 IRQ
- USART1 IRQ

---

## 3.4 Таймер (TIM2)

Поддержать регистры:

- CR1 (CEN)
- PSC
- ARR
- CNT
- DIER (UIE)
- SR (UIF)

Логика:

- CNT увеличивается по internal ticks
- Overflow → UIF
- Если UIE=1 → IRQ

---

## 3.5 USART1

Поддержать:

- SR (TXE, RXNE)
- DR
- BRR
- CR1 (UE, TE, RE, RXNEIE)

### Поведение

TX:
- Запись в DR → вывод в stdout/буфер

RX:
- Данные поступают из job.config.uart_input
- Установка RXNE
- Генерация IRQ

---

## 3.6 GPIO (MVP)

- MMIO-заглушка
- Минимальная реализация чтения входного пина

# 4. Требования к микросервису (Go)

## 4.1 Формат задачи

```json
{
  "job_id": "string",
  "firmware": "base64",
  "config": {
    "max_instructions": 1000000,
    "timeout_ms": 5000,
    "uart_input": "string"
  }
}
```

---

## 4.2 Формат результата

```json
{
  "job_id": "string",
  "status": "OK | TIMEOUT | CRASH | UNSUPPORTED_INSTR | FAULT",
  "exit_code": 0,
  "uart_output": "string",
  "instructions_executed": 12345
}
```

---

## 4.3 Требования к запуску

- Симулятор запускается как отдельный процесс
- Ограничение по времени
- Ограничение по количеству инструкций
- Сбор stdout/stderr

---

## 4.4 Логирование (OpenTelemetry)

Для каждого job:

Spans:
- dequeue_job
- run_simulator
- publish_result

Атрибуты:
- job_id
- status
- instruction_count
- timeout_ms
